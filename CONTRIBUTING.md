# Contributing to Astroxs

Thank you for considering contributing to Astroxs! We welcome contributions from the community.

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue on GitHub with:
- A clear title and description
- Steps to reproduce the issue
- Expected vs actual behavior
- Laravel and PHP versions
- Any relevant code samples or error messages

### Suggesting Features

Feature requests are welcome! Please:
- Check if the feature has already been requested
- Clearly describe the feature and its use case
- Explain why this feature would be useful to most users

### Pull Requests

1. Fork the repository
2. Create a new branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Ensure your code follows PSR-12 coding standards
5. Write or update tests as needed
6. Commit your changes (`git commit -m 'Add amazing feature'`)
7. Push to your branch (`git push origin feature/amazing-feature`)
8. Open a Pull Request

### Code Style

- Follow PSR-12 coding standards
- Write clear, descriptive commit messages
- Comment your code where necessary
- Keep methods focused and concise

### Testing

- Write tests for new features
- Ensure all tests pass before submitting PR
- Maintain or improve code coverage

### Documentation

- Update README.md if adding new features
- Update CHANGELOG.md following Keep a Changelog format
- Add docblocks to new methods and classes

## Development Setup

```bash
# Clone the repository
git clone https://github.com/shrasel/astroxs.git
cd astroxs

# Install dependencies
composer install

# Run tests
composer test
```
